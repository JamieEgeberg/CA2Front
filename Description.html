Description of API, errors and JSON

The REST API schema with errorhandling for person is shown here. Company follows the same template and is therefore not pictured here.

PERSON REST API schema

Path Normal operation Errorhandling
GET
api/person/ Return all Persons, with all details, as a JSON object 500, "No persons found"
api/person/{id} Return the Person (with the given id) with all details as a JSON object 500, "Person with id: {id} not found."
api/person/contactinfo Return all Persons, with only contact info, as a JSON object 500, "No persons found"
api/person/contactinfo/{id} Return the Person (with the given id) with only contact info, as a JSON object 500, "Person with id: {id} not found."
POST
api/person Create a new Person given a JSON object 500, "Persist failed rollback, object: {postObject}"
PUT
api/person Update existing Person given a JSON object 500, "Merge failed rollback, object: {putObject}"
DELETE
api/person/{id} Delete an existing Person given an id 500, "Could not delete. Rollback, object: {toDeleteObject}/null"
